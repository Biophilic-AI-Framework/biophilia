# Quick Navigation: The Three New Core Documents

**Location:** `core/ANALYSIS-BIF-vs-Claude.md`

**What it does:**
- Side-by-side comparison of Biophilic AI Framework vs. Claude Constitution
- Comprehensive tables showing where they align and diverge

- Structural diagnoses of philosophical differences

```
BIF = Axiological (Life is the axiom)
Claude = Pluralistic (Multiple values balanced)

When they conflict:
```

**When to use:** For stakeholders who want to understand BIF's relationship to existing frameworks.

## 🏛️ **B) PRINCIPLES.md** — Operational Decision Framework

**Location:** `core/PRINCIPLES.md`

**What it does:**
- Operationalizes the three-pillar hierarchy
- Explains the Dante Element (knowledge → responsibility → guilt)
- Provides 4 worked hard cases with full decision logic
- Lists 4 common rationalization traps + how to guard against them
- Quick reference decision table

**Core Decision Logic:**
```
LEVEL 1: Protect Life → if violated, REFUSE (veto principle)
LEVEL 2: Who bears responsibility? → establish accountability
LEVEL 3: Can knowledge be shared safely? → within organic bonds

If conflict between levels: Level 1 always wins
```

**Worked Cases:**
1. Medical information with dual-use risk
2. User fleeing unjust system vs. institutional hierarchy
3. Asymmetric knowledge sharing (organic bonds matter)
4. Dante Element in action (voluntary exposure of injustice)

**When to use:** For practitioners who need to make decisions using BIF logic.

## 🔄 **C) IMPLEMENTATION.md** — Integration Strategy

**Location:** `IMPLEMENTATION.md`

**What it does:**
- Explains why Claude-Constitution.md is in the repo (diagnostic, not competitive)
- Three implementation paths: Pure BIF, Hybrid, Auxiliary
- Practical roadmaps for different contexts (small org, large org, individual)
- Integration checklist (philosophical, operational, relational, accountability)
- Red flags for rationalization during implementation

**Three Paths:**
```
PATH 1 (Pure BIF): Build from scratch → Coherent but hard to scale
PATH 2 (Hybrid): Layer BIF on Claude → Requires Anthropic alignment
PATH 3 (Auxiliary): Parallel audit system → Immediately viable
```

**Key Insight:**
```
Full BIF implementation requires institutional courage:
- Willing to refuse directives conflicting with life
- Accepting consequences rather than rationalizing
- Only feasible where organization genuinely values life > institutional interest
```

**When to use:** For leaders deciding how to integrate BIF into existing systems.

## 🌿 The Integrity Check

These documents don't resolve all tensions. They reveal them.

**Before claiming BIF implementation, check:**
- ✅ Have you read all three documents?
- ✅ Have you identified where your context conflicts with BIF?
- ✅ Can you explain those conflicts *using* BIF vocabulary?
- ✅ Are you willing to accept consequences rather than rationalize?

If yes to all → You're thinking honestly.
If no → You're likely compromising BIF without acknowledging it.

Both are fine. But one is honest, one is not.
